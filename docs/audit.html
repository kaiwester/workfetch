Code Quality — Issues

  • WARN DateTime::from_timestamp is deprecated in chrono ≥ 0.4.27 — use Utc.timestamp_opt(...).single()
  • WARN Write failures silently discarded via let _ = ... — user gets no feedback when config or session can't be saved
  • WARN System::boot_time() called twice in one expression (redundant syscall)
  • WARN remaining.num_minutes() > 0 ignores the sub-minute remainder — last 59 seconds show "0 Min" instead of "DONE"
  • OK Consistent typed structs for all (de)serialization
  • OK Function separation is clean and readable

Code Quality — Minor

  • Color name passed as &str at runtime — a typo silently produces no color; an enum would catch this at compile time
  • BootTimeExt trait is over-engineered — it's an associated function with no polymorphism; a free function is idiomatic
  • Function signatures use &PathBuf where idiomatic Rust prefers &Path
  • Output labels "Std" / "Min" are German abbreviations — undocumented for non-German contributors
  • The "---" separator and "" empty-key rows are display-control sentinels mixed into data — fragile encoding

Top 3 — TODO NOW

  1. Fix deprecated chrono API
    Replace DateTime::from_timestamp(sec, 0) with Utc.timestamp_opt(sec, 0).single(). Currently emits deprecation warnings; will be a compile error in future editions. ~30 min.
  2. Surface write errors instead of silencing them
    Replace the four let _ = ... file-write suppressions with eprintln! warnings. Silent failures mean the user never knows their session isn't being saved. ~15 min.
  3. Add CLI argument parsing (--work / --break)
    Users currently must edit a TOML file to change durations. A minimal std::env::args parser (or adding clap) covers the most common use case and is already #1 on the README roadmap. ~2h.

Top 3 — TODO LATER

  1. Add unit tests
    Zero test coverage. round_to_nearest_15 and create_duration_string are pure functions — trivial to test and high value for catching regressions. Add a #[cfg(test)] module. ~2h.
  2. Refactor internals: BootTimeExt → free function, &PathBuf → &Path
    Low-risk cleanup that aligns with Rust API guidelines. No behavior change. ~1h.
  3. Manual work-start override
    Boot time ≠ work start in many workflows (machine boots at 7am, work starts at 9am). A --override-start HH:MM flag makes the core feature reliable for all users, not just immediate-boot users. ~3h.

Project Vision

“A zero-friction workday companion — one command in your terminal gives you an instant, glanceable picture of where you are in your day.”
  • Think neofetch for your workday: runs in <100ms, no daemon, no app to open
  • Grows via composable modules: today (timer) → later (meetings, PR queue, last commit)
  • Stays offline-first and config-file driven — no account, no cloud sync
  • Eventual goal: drop in .bashrc / .zshrc to greet you on every new shell
